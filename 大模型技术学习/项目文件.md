#### 第一阶段：打通底层数据（Day 1 - Day 2）

- **Day 1：监控探针建设。** 用 Python 写脚本，每秒钟去请求一次 vLLM 暴露的 `/metrics` 接口。把长长的英文指标过滤出来，提炼出真正的 GPU 占用、KV Cache 占用比例。
    
- **Day 2：FastAPI 后端骨架。** 建一个 FastAPI 服务，把 Day 1 的探针封装成 WebSocket 接口，为前端实时推送性能数据。
    

#### 第二阶段：构建并发压测引擎（Day 3 - Day 4）

- **Day 3：异步发包机（核心硬核代码）。** 不借助第三方工具，亲手用 Python 协程写一个极速压测脚本。能够在网页指令下发时，瞬间向 8000 端口打出几十上百个长文本对话请求，并精准记录每一个请求的 **发送时间** 和 **收到第一个词的时间 (TTFB)**。
    
- **Day 4：压测任务调度。** 在 FastAPI 中加入任务队列。支持在网页端配置压测参数（如并发数、系统提示词长度、最大输出长度），并将压测过程中的延迟数据实时聚合。
    

#### 第三阶段：可视化大屏与报告生成（Day 5 - Day 6）

- **Day 5：动态仪表盘（前端）。** 写一个单页面 HTML。引入 ECharts，画出三个核心图表：并发请求折线图、KV Cache 实时水位线仪表盘、首包延迟散点图。通过 WebSocket 与后端连通。
    
- **Day 6：自动生成《算力摸底报告》。** 压测结束后，后端自动计算出平均 QPS、95分位延迟（P95 Latency），在网页上弹出一份极具专业感的总结报告。
    

#### 第四阶段：实验室级交付（Day 7）

- **Day 7：打包与守护进程。** 写一个简单的 shell 脚本，让这个网站可以在你的 3090 服务器上长期后台静默运行。以后实验室任何人只要输入服务器 IP，就能看到当前显卡的状态，并在不干扰别人的情况下进行压测。
