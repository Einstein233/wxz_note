**步骤一：静态数字画像注入 (Static Environmental Context Injection)**

- **你的原话**：数学画像提取（固定的）。
    
- **学术内涵**：系统在冷启动时，调用固定的 Python 数据分析流水线，将天津/敦煌的 11 年历史气象数据转化为极其凝练的 JSON 画像（包含总预算、极端应力、光热潜能）。这是整个智能体的**“环境先验知识 (Prior Knowledge)”**。
    

**步骤二：Actor LLM 启发式参数生成 (Heuristic Parameter Generation)**

- **你的原话**：通过 LLM 进行参数确认，然后进行方案生成。
    
- **学术内涵**：LLM 扮演 **Actor（生成器）** 的角色。它读取静态画像后，在巨大的决策空间里进行第一轮“盲猜”（例如：决定分配 30% 给暴雨，光强映射系数设为 1.2）。它输出的是一组结构化的排程策略（Strategy JSON）。
    

**步骤三：Symbolic 引擎物理结算与防幻觉 (Physics-Informed Settlement & Anti-Hallucination)**

- **你的原话**：防幻觉优化。
    
- **学术内涵**：**注意，防幻觉不是靠给 LLM 念咒语，而是靠硬核的数学物理代码！** Actor 生成的参数会立刻被送入 Python 物理引擎。引擎会强制应用我们在 2.2 节定下的数学约束（质量必须守恒、光照能量必须符合平方根等效、强度不能超过 $5.0\text{mm/h}$）。如果 Actor 乱来，这一步会自动进行**“物理边界截断 (Physical Boundary Truncation)”**。
    

**步骤四：多目标评估计算 (Multi-Objective Evaluation)**

- **你的原话**：评估函数计算。
    
- **学术内涵**：排程表在物理引擎里跑通后，系统计算两个核心指标：一个是代表时间成本的 $T_{lab}$，另一个是你刚才彻底弄懂的、代表分布保真度误差的 **NETS（洛伦兹轨迹积分差）**。
    

**步骤五：Critic LLM 语义反馈与迭代 (Semantic Directional Feedback & Iteration)**

- **你的原话**：通过 Evaluate LLM 来进行评估并指导进行迭代优化，最后得到结果。
    
- **学术内涵**：LLM 切换为 **Critic（评论家）** 角色。它读取步骤四算出的冷冰冰的 NETS 和耗时分数，将其转化为**“文本梯度 (Textual Gradient)”**（例如：“当前方案耗时极佳，但 NETS 误差偏大。请在下一次迭代中减少暴雨占比”）。Actor 接收到这段语义指导后，重新生成策略。这个循环最多进行 $K$ 次（例如 3 次），最终输出在物理安全边界内的**帕累托最优排程 (Pareto-Optimal Schedule)**。
