### 宏观流程
构建第K步的预测流程
1、从LTM中检索出相关环境/阶段的规则
2、构建第K步的STM
3、选择是否需要数值工具得到结果
4、进行预测得到下一步的损伤
5、LLM判断是否需要进行反思
6、将损伤加入到已有的损伤序列中，更新状态
7、结束本轮进入下一轮

构建第K步的STM：
**输入（来自 AgentState）：**

- `recent_seq`：最近几步的 env+damage（长度例如 3）
    
- `current_params = env_plan[k]`
    
- `current_damage = D_pred[k]`
    
- `retrieved_rules`：上一节点返回的规则
    
- `env_plan`：全局环境
    
- `current_step = k`
    
- `max_steps = T`
    
- `pred_damage_series`：已预测损伤曲线（0…k）
    
- `tool_results`：如果上一轮已经做过工具计算，这里也可以带着
    

**内部处理：** 就是你贴出来的 `build_stm_for_prediction(...)`：

1. 把 `recent_seq` 规范化成 `short_term_history`（最近几步的结构化 env+damage）；
    
2. 从 `env_plan` 中切出未来 24 步的 `future_env_window`；
    
3. 把当前 env+damage、一点全局信息（current_step、max_steps）、LTM 规则、最近几步预测、工具结果都塞进一个 JSON 里。
    

**输出：**

- `stm_json`：给 LLM 的 JSON 字符串；
    
- `stm_obj`：同样内容的 Python dict，方便后面节点在 Python 里面用。
    

这一步结束后，你可以直观地理解为：

> “我们已经把第 k 步需要知道的一切——  
> 过去几步、现在、短期未来、经验规则、工具结果、自身预测历史——都整理到一张纸上了。”
