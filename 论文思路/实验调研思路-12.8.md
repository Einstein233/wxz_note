# 数据层
环境检测数据：温度、湿度、降雨、风速等时序数据。（多场耦合数据）
实验数据：波速、硬度、色度、含水率、累计产流/产沙
元数据：材料类型、结构信息、地理位置、季节、试验工况

知识：专家经验知识+文献机理探究

# 基于LLM的语义理解与特征提取层
将“多环境要素+材料信息+专家知识”转化为一个高质量、可解释的状态向量Z。

总结：
本研究在预测模型前端构建了一个基于大语言模型与专家知识库的特征表示层，以在缺乏大规模标注样本的条件下充分利用领域知识，实现多环境要素耦合状态的高质量表征。首先，利用预训练大语言模型对原始环境监测数据和材料参数进行语义解析与变量角色标注，自动识别温度、湿度、降雨、风速等环境驱动因子及其在文物损伤过程中的作用。随后，大语言模型基于解析结果构造针对性的检索请求，从专家知识库（包括既有文献、标准规范和项目经验）中提取关于干湿循环、冻融次数、高湿阈值等机理性知识，据此生成一套物理先验特征设计方案，例如累计降雨量、高湿持续时间、干湿/冻融循环次数、危险日较差天数等。  
在实现层面，通过常规程序对该特征方案进行计算，将环境时序数据映射为一系列数值特征，并与材料属性和机理类别等知识标签共同拼接，形成统一的多场耦合状态向量 z，作为后续 LLM+小模型损伤曲线预测模块的输入。整个特征构造过程中不依赖损伤标注数据，仅依托预训练大模型的语义理解能力和专家知识库的记忆，实现了在 0 样本条件下的知识驱动特征提取，为少样本损伤预测与机制解释提供了坚实基础。

1、输入整理&基本预处理
输入内容：环境时序数据（温度、湿度、降雨、光照）、样本的信息（材料类别、是否加固、密度）
2、基础的预处理
时间对齐、单位统一、缺失值填补
3、LLM语义理解+变量角色标注（0样本LLM预测的核心关键能力）
输入：【字段名、单位、简短说明】，各字段的一些统计值【均值、方差、分布范围】
任务：变量角色分类，将每个字段进行分类为【环境因子、材料特性、损伤量】
领域语义标注：给每个字段加上“在损伤过程中如何发挥作用”
		温度：热–力耦合、冻融、蒸发
		降雨：侵蚀
		波速：强度指标
作用：为LLM对每个因子、指标的理解做重要支撑。
4、专家知识库的结合
模型依据数据内容，提出一些问题，根据问题在知识库中进行检索。
示例：“夯土/砂土 遭受干湿循环、冻融、降雨时，常用哪些指标衡量环境危险程度？”
	“文献/标准中如何定义高湿阈值、危险日较差、冻融次数阈值？”
	“在土遗址风化研究中，哪些环境因子被认为是主控因素？”
5、LLM根据检索结果，生成特征提取方案。
即LLM需要根据哪些公式、方法，来提取数据中的哪些特征？
例如，温度特征设计：需要知道日均温度、温差、是否冻融？
湿度特征设计：高湿持续时间、干湿循环次数、平均湿度。
降雨特征设计：累计降雨量、降雨时间次数、最大雨强。
6、按照特征提取方案，进行计算，得到数值特征并向量化。
LLM根据公式计算出现有的特征值后：
已有的数据包括：
静态材料特征【材料类型、密度、是否加固】
环境特征【上述计算所得】
知识标签特征【检索所得的文本数据】
编码为向量。
