K8s:
pod 最小调度单位，可以存放一个或者多个容器，他们共享网络和存储
Deployment 负责控制副本数量
service 稳定的访问入口，pod的IP是会变化的，Service向一个负载均衡器，保证别人总能找到你拉起的服务
NameSpace 逻辑隔离圈 开发环境与生产环境合占一个位置，互不干扰
PV/PVC 云硬盘，容器挂了，数据不能丢，必须挂在外部存储

**k8s基础操作学习**
1、查看现在的领地(nodes)
每个服务器是一个nodes
kubectl get nodes
你会看到一个状态为 `Ready` 的节点。这说明你的“管家”已经就绪，可以接收指令了。
2、发布第一个应用(Deployment)
kubectl create deployment my-wed --image=nginx
这就是拉起自己的第一个应用
kubectl get pods
这样就是将自己的应用部署到pods中，
3、可以一键快速扩容，将我们的应用一建拉起几十个，这样能够增加服务器的压力
kubectl scale deployment my-web --replicas=3
k8s可以自动完成负载的分发与容器的启动
4、如果容器挂了，k8s会直接帮我们进行重启
kubectl delete pod <你的Pod名字>
之后k8s会自动创建一个新的pods
5、pod的IP会随时改变，每个pod都有自己的IP，因此需要一个固定的电话
kubectl expose deployment my-web --type=NodePort --port=80
检查结果
kubectl get svc
这样会对这样3个pod提供一个统一的接口，无论pod如何改变，访问service都可以成功。
